Search problem : Given a graph G=(S,A) while S is the set of vertices and A is the set of edges, a search problem is define with a tuple (G, Si, Sg, T , C)

  • G is mean State space
  • Si is mean the initial state ( e.g. “start point”)
  • Sg is mean the goal state ( e.g. “final destination”)
  • T is mean transition function, the transition function is represent receives a state S and return a set of state S’ such (S , S’) belong to A
  • C is mean cost function, the cost function is represent receives an edge in A and return the cost of the action the edge represents

Search tree : the search tree is a way to solve a search problem, in a search tree there are children and parent. We say that the children of a node n is generated when a search algorithm invokes the transition function for n. A node is expanded when their children are generated.

In search tree, the repeated states in the tree can be of two types: cycles or transposition

  • cycles ( A cycle occurs when the same state appears twice on the same path.)
  • Transpositions (Transpositions occur when multiple paths lead to the same state.)

Uniformed search Algorithm

First in first out (open list)

  • Organized the search in layers

Hash table (close list)

  • Keep track of visited states
  • Recovers the solution path

BFS (pseudo code)

Def BFS(sint, sg, T): 
	open.insert(sint)
	close.insert(sint)
	while not open.empty():
		m = open.pop()
		for m' in T(m):
			if m' is goal sg:
				return path
			is m' is not in closed:
				open.insert(m')
				close.insert(m')
  1. complete
  1. 一定找到最短路径（cost不是最小）
  1. space complexity： b^d
  1. Time complexity： b^d

Dijkstra’s algorithms

  • from smallest length to smallest cost.
    • expand cheapest node in open

    def dijkstra (sint, sg, T)
	open.append(sint)
	close.add(sint)
	while not open.empty()
		n=open.pop()
		if sint==sg:
			return path from sint to n′
		for n′ in T (n): 
			if n′ not in CLOSED:
				OPEN.append(n′ ) 
				CLOSED.add(n′ )
			if n′ is in CLOSED and g(n′) < CLOSED[n′].g_value:
				update g(n′) in OPEN and CLOSED
				update parent of n′ in CLOSED
				heapify OPEN
  • 一定找到cost最小的路径（optimal）
  • 在open中随时更新最小的cost
  • 每次探索选择cost中最小的node
  • 与BFS空间度咋读一样占空间比较多

Important differences

  • stop when goal comes out of open
  • upload cost to nodes if a better path is found
  • FIFO → PQ (Queue to Priority Queue, 从选择最晚的变成选择cost最小的）
